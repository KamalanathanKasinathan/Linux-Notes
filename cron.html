Schedule Task in Linux

What is cron Command?

The cron tool allows Linux users to execute commands or scripts at a specific date and time.

It is very useful for performing regular tasks such as backups, daily scans, /tmp cleanup, and restarting the system at a given time.

Cron

Package Installation

# yum install cronie  (or) crontabs

Service Status

# systemctl status crond

Edit Crontab Entries

crontab -e

Example Cron Jobs

* * * * * script.sh > /tmp/script_out
*/5 * * * *
0 10 * * *
30 1 * * *
30 1,4,5 1-10 * *

Special Time Strings

@daily
@reboot
@hourly
@yearly

List Existing Cron Jobs

crontab -l

Remove Crontab Entry

crontab -r

Edit Crontab for a Different User

crontab -u username -e
crontab -u username -r
crontab -u username -l

General Syntax of a Cron Job

MIN   HOUR   Day of Month  Month  Day of Week  Command
0-59  0-23   1-31         1-12   0-6         Any Linux command or script

Crontab File Locations

/var/spool/cron/user_name
/var/spool/cron
/var/log/cron
/etc/cron.deny
/etc/cron.allow
/etc/cron.hourly
/etc/cron.daily
/etc/cron.monthly

Important Notes

  • Might be affected by different time zones.
  • If the user account and password expire, cron jobs will not work.
  • If the crond service is stopped, cron jobs will not work.
  • If the /var filesystem is 100% full, cron jobs will not work.
