Special Permissions

Special File/Folder Permissions

Types of Special Permissions

  • Setuid (SUID) - 4
  • Setgid (SGID) - 2
  • Stickybit - 1
        suid    sgit    stickybit
        rwx     r-x     r-x
          s      s       t
        rws     r-s     r-t
        rwS     r-S     r-T

Command Examples

  • chmod u+s file/directory
  • chmod g+s file/directory
  • chmod o+t file/directory
  • chmod 4755 file
  • chmod 2755 file
  • chmod 1744 file
  • chmod 0755 file

Setuid

Set User ID (SETUID): Whenever SETUID permission has been set on executable files, anyone executing that command (file) will inherit the permissions of the owner of the file. Its numeric value is 4.

        #chmod 4755 <filename>
        #chmod u+s <filename>
        #chmod u-s <filename>

SGID

Set Group ID (SETGID): Similar to SETUID, but the process's effective group ID (GID) is changed to the group owner of the file, and a user is granted access based on permissions assigned to that group.

        #chmod 2755 <file/dirname>
        #chmod g+s <file/dirname>
        #chmod g-s <file/dirname>

Sticky Bit

If a directory has the sticky bit set, files in that directory can only be deleted by their owner or the root user.

        #chmod 1755 <file/dirname>
        #chmod o+t <file/dirname>
        #chmod o-t <file/dirname>
