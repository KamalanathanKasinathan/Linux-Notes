What is Soft Link?

A symbolic link, also known as symlink or soft link, is a special type of file that points to a source file or directory in Linux.

It is like a shortcut in Windows which contains the path of the original file and not the contents.

    # ln -s <sourcefile>  <dest file>
    # ln -s <sourcefile full path>  <dest file path>

What is Hard Link?

A hard link is a mirror copy of the original file. Deleting the original file will not impact anything, because the hard link file will act as a mirror copy of the original file.

    # ln <source file> <destination file>
    # unlink <file/dir>

Difference Between Soft and Hard Link

Soft Link

  • Soft link is similar to the file shortcut in Windows.
  • Changes in both files will reflect simultaneously.
  • Soft links can be created on different file systems.
  • Soft links can be created for files and directories.
  • It has a different inode number and file permissions compared to the original file.
  • When the original file is removed, the link will be inaccessible (hanging link).

Hard Link

  • Hard link is a mirror copy of the original file.
  • It is like a soft link.
  • Hard links can only be created on the same file system.
  • Only files can be linked.
  • It has the same inode number and file permissions.
  • Nothing happens when the original file is removed.

What Is The Inode Number?

An inode is a data structure that is used to store information of files on your Linux system.

The number of inodes shows the number of files and directories you have on the system.

Every inode in the Linux structure has a unique number identified with it. It is also called the index number and has the following attributes:

  • Size
  • Owner
  • Date/time
  • Permissions and access control
  • Location on the disk
  • File types
  • Number of links
  • Additional metadata about the file

To check the list of inode numbers, use the following command:

    # ls -li
    # stat <file/dir>
